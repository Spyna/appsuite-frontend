Permissions

Permissions define what a user can do with a particular folder and its contents. The single users have defined folder permissions. Depending on the folder type, different permissions are predefined:

  • You have administration rights for your personal folders. This means that you can e.g. create or delete subfolders. Other users do not have any permissions for your personal folders.

  • For the Public folders folder you have the permission to create subfolders and share them with other users.

  • If you want another user to make use of information in one of your private folders, you have to give the user specific permissions for your folder. This is also called sharing the folder.

Before sharing information, please note:

  • You can not share single items, only complete folders.

  • Think about the permissions needed by another user. If a user should only read folder contents, write permission for this folder is not needed. You can not share single items, only complete folders.

A description of defined permissions can be found in : Which permissions are defined?. A listing of preset permissions for specific folders can be found in : Which folder permissions are preset?.

Which permissions are defined?

Note the following:

  • If a user belongs to several groups with different rights for a folder, the user will always get the permission highest in the ranking.

  • There is a permission hierarchy. The permission that is higher in the ranking comprises the permissions that are lower in the ranking.

The following overview always lists the permission highest in the ranking on the bottom. The following permissions exist:

  • Folder admin  Defines whether a user is the folder administrator.

    • No  The user is not the administrator.

    • Yes  The user is the administrator.

  • Folder rights  Defines what a user can do with the folder.

    • None  The user has no rights for the folder.

    • Visible folder  The user can see the folder.

    • Create objects  The user can create objects in the folder.

    • Create subfolder  The user can create subfolders in the folder.

    • Maximum  The user can create subfolders in the folder.

  • Read objects  They define which folder objects are visible to a user.

    • None  The user can not see the objects.

    • Read own  The user can only see the objects created by him/her.

    • Read all  The user can see all objects.

    • Maximum  The user can see all objects.

  • Modify objects  Defines which folder objects can be modified by the user.

    • None  The user can not modify any objects.

    • Modify own  The user can only modify objects created by him/her.

    • Modify all  The user can modify all objects.

    • Maximum  The user can modify all objects.

  • Delete objects  Defines which folder objects can be deleted by the user.

    • None  The user can not delete any objects.

    • Delete own  The user can only delete objects created by him/her.

    • Delete all  The user can delete all objects.

    • Maximum  The user can delete all objects.
